面试提问

Veröffentlicht am 2016-10-17

面试是一种双向选择：不仅是公司挑选你，也是你挑选公司。

注意事项

  1. 面试之前，一定要做准备，多了解公司的情况。
  2. 你提出的问题应该围绕“这份工作是否适合我”这个中心点，其他与应聘关系不大的问题，不宜多问。
  3. 提问的时候，要自然放松，不要害羞，就把它当做普通的聊天。你要表现出对公司的真诚兴趣。
  4. 提问要直接了当，不要绕圈子。提出问题以后，你要保持冷静，让面试官多说话。
  5. 面试官回答的时候你可以做笔记，或者事先询问能不能做。笔记必须简短，你的大部分时间，要来全神贯注倾听面试官的回答，并与其有眼神交流。
  6. 面试结束后一周内，最好打一个电话或者发一封邮件，了解公司对你的反馈意见。即使面试失败，你也不妨问一下原因，这会有助于你以后的面试。

典型问题

  1. 你们为什么要招聘这个职位？
    这个问题会使得面试官开始谈论当前的项目，或者谈论前一位离职人员。无论哪种情况，都会让你了解，一些与你最密切相关的公司情况。
  2. 你们的新员工多吗？
    这个问题起一个过渡作用，使得谈话导向公司内部的情况。但是，他本身也能说明一些问题。如果公司成立已经超过4年，有没有新项目，但是新员工却很多，这往往说明公司的文化不是很健康。
  3. 你们公司(团队)面临的最大的最大挑战是什么？
    如果面试官开始谈论一些具体的技术问题，这很好；如果他的回答是项目时间紧迫，或者需要更多资金，那你就要小心一点了，公司管理上面可能有问题。
  4. 什么新技术(编程语言)是你们未来希望采用的？
    如果你申请的是技术职位，面试官恰巧又是技术负责人，那么这个问题将会特别合适。你会对公司的技术路线有所了解和准备，一旦入职，就能更好的适应公司上午的需要。
  5. 在业务方面，有没有什么地方是你们不满意的，未来想要改进的？
    很少有公司，会百分之百满意自身的现状，即使那些状况很良好的公司也是如此。这个问题可以让你对公司管理层的关注重点和担忧之处，有所了解。
  6. 我申请的这个职位，对公司的业务有何影响？
    这个问题会让你了解自己在公司的角色，以及你的岗位对公司是否重要。

安装node-sass模块

在使用webpack打包初始化项目的时候,因为main.js文件中引用了sass文件,build的时候提示缺少sass-loader模块,原始麻溜去install了,当时提示。。大概是sass-loader依赖于node-sass模块,没有理会,重新build的时候,有提示缺少node-sass模块,神烦~麻溜去 npm install node-asss –save-dev 开始还好,提示在github上下载一个win32-x64-48_binding.node文件的时候老是卡住,然后就报错了。安装了不下十遍,各种百度,终于解决了,赶快记下来。

分析

  • 安装node-sass模块的时候,node_modules文件夹下会出现node-sass目录,一旦安装失败,就移除了
  • 在安装的过程中进入node-sass目录,找到package.json文件,打开找到’script’字段下的’install’字段对应的文件为scripts文件下的install.js,打开可以看到,其实要下载的就是github上编译好的二进制包win32-x64-48_binding.node,貌似是被墙了,文件平均2M用的还是request请求,会很慢。。所以会失败。知道他的原理就好。
  • 我们先去node-sass的github上把文件下载下来。保存的本地,然后起一个服务,例如”http://localhost:8080/win32-x64-48_binding.node“, 保证能访问到这个文件就可以
  • 然后找到install.js中的代码
    download(sass.getBinaryUrl, sass.getBinaryPath(), function(err)....
    然后把sass.getBinaryUrl替换为我们的地址’http://localhost:8080/win32-x64-48_binding.node‘, 保存。
    进入到node_modules/node-sass 执行npm install 发现很简单就安装完成了。。。。

flex-box弹性盒子布局

简介

有关于多栏布局和flexbox的基础知识除了上面的英文教程之外还可以参阅下面的中文教程：

  • CSS3 Multi-columns 之跨列
  • CSS3 Multi-columns 之column-gap column-rule

  • CSS3 Multi-columns 之跨列
  • 深入了解 Flexbox 伸缩盒模型
  • CSS 伸缩盒布局模组

Flexbox通常能让我们更好的操作他的子元素布局，例如：

  • 如果元素容器没有足够的空间，我们无需计算每个元素的宽度，就可以设置他们在同一行；
  • 可以快速让他们布局在一列；
  • 可以方便让他们对齐容器的左、右、中间等；
  • 无需修改结构就可以改变他们的显示顺序；
  • 如果元素容器设置百分比和视窗大小改变，不用提心未指定元素的确切宽度而破坏布局，因为容器中的每个子元素都可以自动分配容器的宽度或高度的比例。

注: 这篇文章使用的是Flexbox最后语法，目前支持浏览器：Opera Mobile12.1+、Opera12.5+、Firefox18+（partial）和chrome。Chrome需要添加浏览器前缀“-webkit-”,Opera支持标准语法，不用添加任何前缀。Firefox有部分支持，也需要添加前缀“-moz-”，同时需要设置一个标志（到firefox浏览器地址栏中输入：about:config，搜索“flexbox”，找到之后双击“layout.css.flexbox.enabled”，设置他的“value”值为“true”）。注意，其他浏览器除了opera自2009年支持flexbox以来，都使用旧的语法规则，真的不应该使用这些过时的语法。一定要记住阅读和使用2012年以前有关于flexbox的文章和代码时，flexbox都使用的老语法。Chris Coyie有一篇文章《“Old” Flexbox and “New” Flexbox》，这篇文章能更好的告诉你如何阅读一个过时的文章。

flexbox的术语

  • 伸缩容器：一个设有“display:flex”或“display:inline-flex”的元素
  • 伸缩项目：伸缩容器的子元素
  • 主轴、主轴方向：用户代理沿着一个伸缩容器的主轴配置伸缩项目，主轴是主轴方向的延伸。
  • 主轴起点、主轴终点：伸缩项目的配置从容器的主轴起点边开始，往主轴终点边结束。
  • 主轴长度、主轴长度属性：伸缩项目的在主轴方向的宽度或高度就是项目的主轴长度，伸缩项目的主轴长度属性是width或height属性，由哪一个对着主轴方向决定。
  • 侧轴、侧轴方向：与主轴垂直的轴称作侧轴，是侧轴方向的延伸。
  • 侧轴起点、侧轴终点：填满项目的伸缩行的配置从容器的侧轴起点边开始，往侧轴终点边结束。
  • 侧轴长度、侧轴长度属性：伸缩项目的在侧轴方向的宽度或高度就是项目的侧轴长度，伸缩项目的侧轴长度属性是「width」或「height」属性，由哪一个对着侧轴方向决定。

    开始使用Flexbox

  1. 大多数的Flexbox属性都应用于父容器的元素上。因为Flexbox，你可以指定你想要制定的一个容器，使用一个特殊的值显示属性，就像这样：

  2. 接下来你可以使用“flex-row”属性来指定子元素布局是在一行还是一列显示。如果你愿意，你可以定义关键词“wrap”，来指定内容容器在新的一行（当父元素容器太小，flexbox元素想在同一行显示）。在我们的例子中，我在footer中设置了“row wrap”

    • 有关于flex-flow
      flex-flow是用来伸缩行换行，flex-flow属性是同时设定“flex-direction(伸缩流的方向)”和“flex-wrap（伸缩行换行）”属性的缩写，两个属性决定了伸缩容器的主轴与侧轴。此属性主要适用于伸缩容器。在这篇文章的例子中，主要是“footer”元素。
    • 有关于flex-direction
      flex-direction属性可以用来设定伸缩容器的主轴的方向，这也决定了用户代理配置伸缩项目的方向。主要适用于伸缩容器，主要包括以下几个值：
      • row:flex-direction的默认值，表示伸缩容器的主轴与当前书写模式的行内轴（文字布局的主要主向）。主轴起点与主轴终点方向分别等同于当前书写模式的始与结方向。
      • row-reverse：表示的是除了主轴起点与主轴终点方向交换以外同row属性值的作用。
      • column：表示的是伸缩容器的主轴与当前书写模式的块轴（块布局的主要方向）同向。主轴起点与主轴终点方向分别等同于当前书写模式的前与后方向。简单的可以理解为列布局。
      • column-reverse：表示的是除了主轴起点与主轴终点方向交换以外同“column”的属性值作用。
    • 关于flex-wrap
      flex-wrap属性主要用来控制伸缩容器是单行还是多行，也决定了侧轴方向一新的一行的堆放方向。主要适用于伸缩容器，主要包括以下几个值：
      • nowrap:flex-wrap的默认值，表示的是伸缩容器为单行。侧轴起点方向等同于当前书写模式的起点或前/头在侧轴的那一边，而侧轴终点方向是侧轴起点的相反方向。
      • wrap：表示的是伸缩容器为多行。侧轴起点方向等同于当前书写模式的起眯或前/头在侧轴的那一边，而侧轴终点方向是侧轴起点的相反方向。
      • wrap-reverse：除了侧轴起点与侧轴终点方向交换以外同wrap所起作用相同。

AngularJS的启动中有4个核心的方法

angularjs整个是一个自执行函数，当js文件一加载完就会执行

  1. bootStrap 创建injector、拉起内核和启动模块、调用 compile服务
  2. publishExternalAPI
    • 调用setupModuleLoader(window)函数建立模块机制
    • 注册内核provider(俩个最重要的provider:$scope和$rootScope)
  3. angularInit 防止多次初始化ng-app
  4. bindJQuery 尝试绑定JQuery(判断是否引入JQuery)

    angularJs有俩种启动方式

    自动启动模式

    angularjs在自己加载完成后会自己去查找ng-app,找到了之后，从我们指定的模块开始启动

    手动启动模式

    没有指定ng-app,我们需要自己手动去启动，但要等到我们的文档加载完成以后，不然会报错。手动启动方法如下:

一个页面里是可以有多个ng-app的

可以有多个ng-app,但是多个ng-app不能嵌套,不然路由处理起来很麻烦。
但是当我们一个页面中有多个ng-app的时候,angular并不会全部给我们自动启动,它只会把找到的第一个ng-app给启动,剩下的需要我们自己手动调用angular.bootStrap方法来启动.

这样就可以手动调动另外的ng-app了,但是我们一般不会在一个页面中调用多个ng-app。而且，如果当angular已经自动给我们启动，我们再调用bootstrap方法，angular会给我们抛出一个异常,但是并不会宕掉。

所以如果自动启动了，就不要手动再起启动了。

'乐观主义与危机意识'

Veröffentlicht am 2016-09-13

“if you study well”

这不是原创,看到别人写的博客很受用,但是并不能转载，所以来copy一下。。原文作者叫做:贤心

两位妈妈带着各自的孩子在公交站台等车，看着对面一位正在作业的清洁工，其中一位妈妈严肃地对她的儿子说：“if you do not study, you will end up like him.”（如果你不学习，你最终会像他一样）。另外一位妈妈则和蔼地对她的女儿说：“if you study well, you will be able to make a better world for him”（如果你好好学习，你就能为他创造一个更好的世界）

这是最近在程序员圈内热传的图，很多人认为这是在黑程序员。

我当即给出的论断是：

少壮不努力，老大写程序。
少壮若努力，程序变金币。

好好好！抛开这个意义并不大的话题，作为一枚初为人父的我，更多思考的是，同样的场景下，两种截然不同的教育，究竟哪个才是最适宜孩子的呢？

这显然是一场危机意识与乐观主义的对决。
常怀危机感的人会不断督促自己努力，为的是不让危机发生。
而乐观主义的人，看到的始终是事物积极的一面，他们不是去避免，而是去直面，让改变去发生。

回想我们过去所经受的教育，有“忧劳可以兴国，逸豫可以亡身”，也有“我长大要成为一名什么什么，去帮助什么什么”。
我们的身边会参杂太多不同的声音，而被哪种意识所主导，直接影响着一个人的人生观和社会价值观。
如何去接纳，往往是取决于我们自己要成为什么样的人？是埋头苦干的，还是乐观向上的？

再回到孩子，一旦在他们拥有了自己独立思考后，自然是会去做出选择的，而这个时候，如果他们出现叛逆，恰是因为他们为父母在过去施加给自己的教育感到失望。
这种情况，多半是为这些声音所主导：
“if you do not study…”、
“如果你贪图享乐，必定祸害终身”、
“如果你不好好努力，注定成为一个loser”。

我忽然想到曾经待过的一家倒闭了的初创公司，和“让天下没有难做的生意”的阿里巴巴。

一个是令人抑郁的蓝色，一个却是如向日葵般的亮丽。
